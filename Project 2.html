Project Summary

This project was a class project for CS 2400 Intro to AI where we chose our own AI algorithm to research about and present to the class. Our task included preparing a 15-minute presentation as well as a code demo of the Hidden Markov Model which we would demonstrate how the model worked using anexample scenario for it.

The final deliverable was the code of the Hidden Markov Model which we modified from a website and the PowerPoint that we presented with. This project consisted of 2 weeks of research, implementation, and practice presenting. Time was a constraint in this project since we couldn’t get too in depth in the material and create the whole model from scratch which led us to use a hmm library in python. This lab was a pair project which helped in being able to cover more material that wouldn’t be possible to if it was an individual lab. My role in this included half the presentation and half the code demo which was implemented by both of us.

Project Analysis

Things I did well in this lab included having an informative presentation with good discussion, examples, and reasons that the model would be used in a scenario. I also believe the presentation was structured nicely so that people could follow the presentation without much issue. This included having the explanation and terms associated with this Hidden Markov Model were toward the beginning so that when the code demo was reached, things would make more sense on how the ideas used to create a HMM (Hidden Markov Model) were implemented in code. I also think I did well in collaborating with my partner since we both were able to finish our portions of work in a timely manner which allowed us to have more time to practice the presentation.

I was able to learn about Hidden Markov Models which I didn’t know about before this project as well as get more practice with Python which included using libraries such as NumPy. From the content learned in the class, I also was able to fit training data to the model and evaluate the effectiveness of each training set. I also got more practice with formatting the code to be presentable and walking through a code implementation which included what each variable was, what outputs represented, and what libraries were used. Improvements I could make for my next project based from this would be to create a git project to be able to have the code collaboration asynchronous if we need to. I would also organize my code more nicely to be well documented and some functions placed in appropriate methods.
